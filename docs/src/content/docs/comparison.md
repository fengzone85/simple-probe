# 竞品对比

> 本页不排座次，只记录设计取向与适用边界。每个工具都有其适合的场景。

## 四方对比

| 维度 | Simple Probe | 哪吒 | Komari | Pulse |
|---|---|---|---|---|
| **定位** | 纯状态监控 | 监控 + 远程控制 | 监控 + 远程控制 | 监控 + 轻量控制 |
| **指令通道** | ❌ 无 | ✅ 有（Web 终端、计划任务） | ✅ 有（远程命令执行） | ❌ 无 |
| **Agent 监听** | ❌ 不监听 | ✅ gRPC 端口 | ✅ 入站端口 | ✅ 入站端口 |
| **Agent 语言** | Python (~200行) | Go | Go | Go |
| **前端** | 原生 JS | Vue | 模板 | Astro |
| **存储** | SQLite | MySQL/PostgreSQL | SQLite | bbolt |
| **2FA** | ✅ TOTP 强制可选 | ✅ | ✅ | ❌ |
| **指纹采集** | ❌ 无 | ✅ 有 | ✅ 有 | ✅ 有 |
| **部署复杂度** | 低 | 中 | 低 | 低 |
| **社区成熟度** | 新项目 | 成熟（10k+ star） | 成熟 | 新项目 |

## 各工具的优点

### Simple Probe
- 零指令通道，架构层面消除 RCE
- 代码量极小（< 800 行），可人工审计
- 原生部署内存 12-25MB，极低占用
- TOTP 2FA、CSP、RBAC 等安全措施齐全

### 哪吒
- 功能最丰富（SSL 监控、Web 终端、计划任务）
- 社区最大，主题生态成熟
- 支持多数据库后端
- 文档完善

### Komari
- 一键部署体验好
- 支持远程命令执行
- 主题可扩展
- 有 Agent 自动发现

### Pulse
- Go 单二进制，部署极简
- bbolt 单文件 DB，零运维
- 自动更新客户端
- 一键迁移/备份

## 选型建议

### 选 Simple Probe 如果你
- 🔐 是安全偏执型运维，不接受任何远程控制能力
- 💾 有小内存机器（256-512MB），需要极低资源占用
- 🕵️ 注重隐私，数据完全自主
- 📋 有代码审计/合规需求

### 选哪吒如果你
- 需要 Web 终端 / SSH 远程管理
- 需要 SSL 证书 / 网站监控
- 想要丰富的主题和插件生态
- 需要多数据库支持

### 选 Komari 如果你
- 需要远程命令执行
- 想要 Agent 自动发现
- 喜欢 1Panel 生态

### 选 Pulse 如果你
- 偏好 Go 单二进制部署
- 想要极简的 bbolt 存储
- 需要客户端自动更新

## 关于 Komari 的安全提示

Komari 支持远程命令执行，其 Agent 已被 Huntress 安全研究机构报告指认为 C2（Command & Control）滥用工具。

> 引用：Huntress 对 Komari Agent 被 C2 滥用的分析报告

这不是说 Komari 本身有漏洞，而是其设计包含指令通道，攻击者可以利用该通道将 Agent 变为 C2 控制端。Simple Probe 通过架构层面消除指令通道来规避此类风险。

## 总结

| 需求 | 推荐 |
|---|---|
| 纯监控、零信任 | Simple Probe |
| 监控 + 远程控制 | 哪吒 |
| 监控 + 轻量控制 | Komari |
| 极简 Go 部署 | Pulse |

每个工具都是其设计取舍下的合理选择。
