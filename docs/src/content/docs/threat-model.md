# 威胁模型

## 三种攻破场景

### 场景 1：服务端主机被攻破

**攻击者获得服务端 VPS 的 root 权限。**

| 资产 | 影响 | Simple Probe 的防护 |
|---|---|---|
| 监控数据 | 泄露（CPU/内存/流量等） | 数据无可利用价值 |
| Agent Token | 泄露 | Token 哈希存储；重置后旧 Token 失效 |
| 其他 Agent | **不受影响** | Agent 仅出站，服务端无法下发指令 |
| Agent 主机 | **不受影响** | 无指令通道，攻击者无法横向移动 |

**结论**：损失限于服务端自身的监控数据，不影响任何被控端主机。

### 场景 2：单个 Agent 主机被攻破

**攻击者获得某台被控端的 root 权限。**

| 资产 | 影响 | Simple Probe 的防护 |
|---|---|---|
| 该 Agent 的 Token | 泄露 | 服务端可重置该 Token |
| 其他 Agent | **不受影响** | Agent 之间零耦合 |
| 服务端 | **不受影响** | Agent 无法向服务端注入指令 |
| 其他被控端 | **不受影响** | 无横向移动路径 |

**结论**：损失限于该被控端自身（已被攻破），不扩散。

### 场景 3：网络流量被劫持

**攻击者能监听/篡改 Agent 与服务端之间的流量。**

| 攻击 | 效果 | Simple Probe 的防护 |
|---|---|---|
| 窃听 | 仅看到加密后的 HTTPS 流量 | TLS 加密 |
| 篡改上报数据 | 服务端收到错误监控数据 | HTTPS + Token 鉴权 |
| 伪装服务端 | Agent 拒绝连接 | HTTPS 证书校验 + 白名单 |
| 重放请求 | 服务端拒绝 | Token 哈希 + 时间窗口 |

## 与哪吒的信任边界对比

| | Simple Probe | 哪吒 |
|---|---|---|
| Agent 监听端口 | ❌ 不监听 | ✅ gRPC 端口 |
| 服务端可下发指令 | ❌ 不可 | ✅ 可（Web 终端、计划任务） |
| Agent 被攻破后影响 | 仅自身 | 可作为跳板攻击服务端 |
| 服务端被攻破后影响 | 仅监控数据 | 可向所有 Agent 下发命令 |

## 诚实的局限性

Simple Probe 不是银弹：

- **hostname/os 入库为轻量标识** — 用于区分节点，非敏感信息
- **Token 哈希降低但不消除明文泄露风险** — 数据库被获取后可离线爆破
- **无指令通道 = 无远程管理能力** — 这是设计取舍，不是缺陷
- **SQLite 单文件 = 无分布式能力** — 适合中小规模（< 100 节点）
