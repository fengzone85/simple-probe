# 安全设计

Simple Probe 的安全模型建立在三个支柱之上。

## 三大支柱

### 1. 无指令通道

受控端（Agent）**不监听任何端口**，仅向服务端发起出站 HTTPS 请求。
服务端没有任何 API 可以向 Agent 下发指令。

**这意味着**：即使服务端被完全攻破，攻击者也无法通过 Agent 执行任何操作。

### 2. Agent 零耦合

Agent 之间互不通信，互不知道彼此存在。
一个 Agent 被攻破不会影响其他 Agent。

### 3. 数据最小化

采集仅限系统状态指标（CPU/内存/磁盘/流量/温度等），**不包含**：
- 进程列表
- 系统配置
- 网络连接
- 用户信息
- 密钥/凭据

采集数据对攻击者无可利用价值。

## 安全措施清单

### 鉴权与会话

| 措施 | 说明 |
|---|---|
| TOTP 2FA (RFC 6238) | 管理员登录需动态验证码 |
| 恒定时间比较 | Token 比较 `SHA-256 + timingSafeEqual`，消除长度侧信道 |
| HttpOnly Cookie | Session Cookie 不可被 JS 读取 |
| 登录限速 | 防暴力破解 |
| 2FA 重置 Token | 一次性使用，有效期可配 |

### 受控端加固

| 措施 | 说明 |
|---|---|
| HTTPS 强制 | 非 localhost 请求 `exit(1)` |
| 401/403 长退避 | 认证失败退避 600 秒，防爆破 |
| 非运行 | Docker: `--user 65534` + 只读挂载 |
| systemd 14 项加固 | NoNewPrivileges、ProtectSystem、ProtectHome 等 |
| 无 shell 调用 | `probe_one()` 使用 list 形式命令，不经过 shell |
| 本地固定探测目标 | 网络质量测试目标来自本地环境变量，服务端不可下发 |

### 传输安全

| 措施 | 说明 |
|---|---|
| HTTPS 白名单 | 受控端仅信任配置的 Server URL |
| CSP 头 | 服务端设置 Content-Security-Policy |
| Token 哈希存储 | 数据库存储 SHA-256 哈希，非明文 |

### 前端安全

| 措施 | 说明 |
|---|---|
| XSS 防护 | 所有用户输入经转义 |
| CSRF 防护 | Cookie + SameSite |
| 输入校验 | `validate.js` 统一校验 |

## 数据保留

服务端每小时自动清理过期指标，防止数据库无限膨胀。保留天数可在「设置 → 告警规则」配置（7-3650 天），也可通过环境变量 `RETENTION_DAYS` 设置（默认 30 天）。后台设置优先级高于环境变量，保存后 1 小时内自动生效，无需重启。

## 数据备份

数据库是监控系统的核心资产（含全部 Agent 记录、历史指标、设置），建议定期备份：

```bash
# 一键备份（热备份，不中断服务）
sudo bash install.sh --backup

# 定时自动备份（crontab）
0 3 * * * root bash /usr/local/bin/simple-probe-install.sh --backup
```

备份文件默认存到 `/var/backups/simple-probe/`，包含完整数据库快照。恢复前会自动备份当前状态，确保可回滚。

## TOTP 2FA

Simple Probe 支持 TOTP（基于时间的一次性密码），兼容 Google Authenticator、1Password 等。

### 启用流程

1. 登录管理面板
2. 进入设置 → 安全
3. 扫描二维码或手动输入密钥
4. 输入 6 位验证码确认

### 2FA 重置

如果丢失 2FA 设备，可通过一次性重置 Token 恢复。重置 Token 在启用 2FA 时生成，请妥善保管。

## 信任边界

```
┌─────────────────────────────────────────┐
│  服务端 VPS                              │
│  ┌───────────┐    ┌──────────────────┐  │
│  │  Agent A  │───▶│                  │  │
│  └───────────┘    │   Server (Node)  │  │
│  ┌───────────┐    │   + SQLite       │  │
│  │  Agent B  │───▶│                  │  │
│  └───────────┘    └──────────────────┘  │
│  ┌───────────┐                          │
│  │  Agent C  │───▶ (只出站，不监听)      │
│  └───────────┘                          │
└─────────────────────────────────────────┘

信任边界：每个 Agent 独立，互不通信
攻击面：仅服务端 Web 端口
```
